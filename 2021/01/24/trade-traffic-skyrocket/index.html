那些年在商城奋斗的光辉岁月 - 交易系统-下单流量暴涨

发表于 2021-01-24 更新于 2023-03-12 分类于 stability 阅读次数： 阅读次数： Valine：
本文字数： 3.7k 阅读时长 ≈ 3 分钟

背景 随着业务的快速发展，每天交易的订单量已经过千万，从2021-01-20开始，交易系统-下单服务的QPS几乎每天翻一倍，随时可能触发限流阈值，影响用户正常下单。

(1) 问题描述

在业务快速发展的时候，日订单量达到千万单，忽然交易系统-下单的流量翻倍，而且每天翻倍，但是每天的成交订单量确几乎没有变化，比较奇怪；

更严峻的问题是如果流量继续翻倍，交易系统-下单服务可能触发限流，影响正常的交易，这个是都不愿意看到的。

(3) 问题分析

  1. 下单时触发支付限流(创建订单成功，创建预付单失败)，下单报错，对用户提示“调用收银台返回异常”
  2. 用户继续点击，触发限购，对用户提示“秒杀商品xx，您的购买已经超过上限”
  3. 用户继续点击，触发下单限流，对用户提示“请勿重复提交”

下单支付流程

(3.1) 问题直接原因

  1. 下单时收银支付接口限流导致下单失败(创建订单成功-生成预付单失败)。
  2. 前端交互存在问题，服务端下单接口返回错误码，没有跳转到对应页面，用户重复点击，导致流量翻倍。
  3. 前端在用户点击下单支付按钮后，没有把按钮置灰。
  4. 服务端按照用户维度限流时间较短。

(3.2) 问题详细原因

(3.2.1) 为什么下单流量高了？

  1. 同一天同一秒杀活动不同(时间)场次的被限购，用户多次重试。
  2. 优选业务处在快速增长时期，一直在开城，流量每天增高是正常现象。
  3. 问题发生事件在年底(腊月初八 ~ 腊月十二)，用户在备年货，运营在搞活动，团队在冲业绩，商品活动和用户活动较多。
  4. 秒杀商品价格低于成本价，导致购买用户较多，瞬时交易流量较大。
  5. 下单失败用户重试导致流量翻倍。

(3.2.2) 为什么收银支付触发限流？

  1. 下单流程里包含创建订单和创建收银支付单。
  2. 下单的QPS高了导致收银支付的QPS也高了。
  3. 当时只接了收银支付的一个机房，QPS达到收银支付那个机房的阈值，触发限流。

(3.2.3) 为什么前端交互存在问题？

按照预期的流程，用户点击立即支付按钮后，前端调服务端下单接口，下单创建订单成功，生成预付单失败，返回了错误码，前端根据错误码跳转到订单列表提示用户二次支付；
但实际没有跳转。

  1. 历史原因，前端不知道有个错误码没有处理。
  2. 其它错误码都处理了，也可能是前端搞出的bug。 (前端合代码冲突 或 误删代码)
  3. 在微信小程序交易场景，没有收银台，默认支付方式微信支付，前端调完下单接口即调用微信支付SDK，这样用户体验好，交易效率也高。

(3.2.4) 为什么用户会重复点击？

  1. 秒杀活动商品价格低于成本价，用户都想抢，迟了可能会售罄。
  2. 服务端下单接口返回错误码和错误信息后，前端处理成提示了。
  3. 前端没有把立即支付按钮置灰，用户可以一直点击。
  4. 服务端下单接口幂等时间较短。

(4) 思考

(4.1) 为什么忽然出现流量暴涨的问题？

下单流量一直在上涨，以前怎么没有出现类似翻倍的情况？

(4.2) 为什么01-20下单流量忽然暴涨？

  1. 支付限流
  2. 下单失败后前端没有跳转，用户重试。

(4.3) 为什么01-22下单QPS又翻倍？

01-21秒杀有个重构，正好01-21上线，可能和同一天支持多个

  1. 秒杀支持同一天同一个秒杀活动配置不同(时间)场次，加上优惠力度比较大，吸引很多用户一天多次购买。
  2. 支付(创建与付单)限流
  3. 下单(创建订单)限流
  4. 下单失败后返回错误码，前端拿到错误码没有跳转。
  5. 用户重复点击立即支付按钮，下单(创建订单+预付单)流量翻倍
